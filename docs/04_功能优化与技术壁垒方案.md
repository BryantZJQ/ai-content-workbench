# AI短视频内容工作台 — 功能优化与技术壁垒方案

> 基于全量代码审查后的深度分析
> 目标：从"能用的工具"进化为"离不开的平台"
> 最后更新：2026-04-03

---

## 一、当前功能全景与差距分析

### 1.1 现有 8 大功能

| # | 功能 | 核心能力 | 壁垒等级 | 短板 |
|---|------|----------|----------|------|
| 1 | 热搜榜单 | 4平台并发抓取 | ⭐ 低 | 纯抓取，任何人都能做 |
| 2 | 智能选题 | 热搜+AI联想+打分 | ⭐⭐ 中 | 一次性结果，无法迭代 |
| 3 | 脚本生成 | 12风格+4平台 | ⭐⭐ 中 | 生成完就结束，无优化闭环 |
| 4 | 分镜提示词 | 中英双语提示词 | ⭐⭐⭐ 中高 | 与可灵/Pika无API直连 |
| 5 | 一键出片 | 选题→脚本→分镜全自动 | ⭐⭐⭐ 中高 | 流水线单向，无法回退修改 |
| 6 | 爆款拆解 | 结构公式+套用新选题 | ⭐⭐⭐⭐ 高 | 公式用完即弃，无法沉淀 |
| 7 | 标题优化 | 生成+对比+评分 | ⭐⭐ 中 | 与脚本生成割裂 |
| 8 | 脚本诊断 | 六维评分+完播率预测 | ⭐⭐⭐ 中高 | 诊断后只能看，不能一键修 |

### 1.2 核心差距

**当前最大问题：功能之间是"孤岛"，用户用完一个Tab要手动复制到另一个Tab。**

- 诊断出问题 → 无法自动修复
- 拆解出公式 → 无法沉淀复用
- 选题→脚本→分镜 → 中间无法回退调整
- 所有生成结果 → 关掉页面就丢了

这意味着：用户可以被任何同类工具替代，没有"迁移成本"。

## 二、技术壁垒级优化方案（按壁垒强度排序）

---

### 🔴 壁垒等级 S — 数据飞轮（最难复制）

#### S1. 「诊断→一键重写」闭环

**现状痛点：** Tab8诊断出脚本弱点后，用户只能看建议，手动回Tab3重新生成。

**优化方案：**
- 诊断结果页增加「🔧 一键优化脚本」按钮
- 将诊断的具体问题（弱点、建议）注入到 LLM prompt，要求**仅修改弱项，保留优点**
- 输出"优化前 vs 优化后"对比视图
- 优化后可再次诊断，形成 **诊断→重写→再诊断** 的迭代循环

**技术壁垒：** 
- 多轮AI调用的上下文串联（当前脚本 + 诊断结果 + 重写 prompt）
- Prompt链路越深，竞品越难抄
- 用户体验到"越用越好"，形成粘性

**实现难度：** ⭐⭐（中等，主要是prompt工程 + 前端交互）

**代码改动点：**
- `content_analyzer.py` 新增 `rewrite_script(original, diagnosis)` 函数
- `app.py` Tab8 结果区增加重写按钮和对比视图

---

#### S2. 「爆款公式库」沉淀系统

**现状痛点：** 用户用爆款拆解提取了公式，但关掉页面就丢了。下次要重新拆解。

**优化方案：**
- 拆解完成后增加「⭐ 收藏到我的公式库」按钮
- 公式库存入 `session_state` 或 Supabase（按卡密关联）
- Tab3脚本生成增加「📚 从公式库选模板」下拉
- 一键出片也可选「用xx公式风格生成」

**技术壁垒：**
- 用户积累越多公式 → 越不愿离开
- 公式库本身就是竞品不具备的"数据资产"
- 后期可做"热门公式排行榜"（社区化方向）

**实现难度：** ⭐⭐⭐（需要存储层改造，Supabase新增公式表）

---

#### S3. 「创作历史」记忆系统

**现状痛点：** 刷新/关闭页面，所有生成内容全部丢失。

**优化方案：**
- 所有生成结果自动存入 Supabase（按卡密关联）
- 侧边栏增加「📂 我的创作」折叠面板
- 支持：查看历史 / 继续编辑 / 重新诊断 / 导出全部

**技术壁垒：**
- 用户积累的创作内容成为"沉没成本"，迁移成本极高
- 后期可分析用户偏好，做个性化推荐
- 真正的"平台"而非"工具"

**实现难度：** ⭐⭐⭐⭐（Supabase schema设计 + 前端历史列表 + 数据迁移）

**建议阶段：** Phase 2（优先做S1和S2，见效快）

---

### 🟡 壁垒等级 A — 工作流壁垒（较难复制）

#### A1. 跨Tab智能联动（打通功能孤岛）

**现状痛点：** Tab之间靠 `session_state` 传递，但联动不够深。

**具体优化：**

| 联动路径 | 当前状态 | 优化后 |
|----------|----------|--------|
| 选题 → 脚本 | ✅ 有（expander选择） | 选题结果直接作为脚本生成的下拉选项 |
| 脚本 → 分镜 | ✅ 有（checkbox复用） | OK |
| 脚本 → 诊断 | ❌ 没有 | 脚本生成后增加「🔍 立即诊断」按钮 |
| 诊断 → 重写 | ❌ 没有 | 见S1方案 |
| 爆款拆解 → 脚本 | ✅ 有（套用公式） | 增加公式库沉淀（S2） |
| 标题 → 脚本 | ❌ 没有 | 标题生成后可「用此标题重新生成脚本」 |
| 热搜 → 选题 | ⚠️ 间接 | 热搜条目增加「📌 加入选题种子」按钮 |

**技术壁垒：** 完整的工作流闭环让用户感觉"所有事在一个地方搞定"，竞品即使功能相同，缺少联动就体验差很多。

**实现难度：** ⭐⭐（主要是前端交互和session_state传递）

---

#### A2. 「内容周历」批量规划器

**现状痛点：** 每次只能生成1个选题/脚本，做内容矩阵的用户需要重复操作。

**优化方案：**
- 新Tab或一键出片的扩展模式
- 输入：赛道 + 一周发布频率（如每天1条）
- 输出：7天选题日历，每天配不同选题角度，自动避免重复
- 每个选题可一键展开生成脚本

**技术壁垒：**
- 竞品大多只做单次生成，"批量+去重+角度分散"是独家体验
- 符合专业创作者的真实工作流

**实现难度：** ⭐⭐⭐

---

#### A3. 「平台算法适配」深度优化提示

**现状痛点：** 平台选择只影响分镜画面比例，脚本内容并未针对各平台算法做差异化。

**优化方案：**
- 抖音：强化前3秒钩子 + 争议性话题 + 完播率优先
- B站：知识密度 + 个人观点 + 长尾SEO
- 小红书：标题党 + 情绪价值 + 关键词种草
- 在脚本生成 prompt 中注入平台专属规则
- 诊断时按平台算法特点给出差异化建议

**技术壁垒：** Prompt工程的know-how积累，竞品不理解平台差异就无法复制效果。

**实现难度：** ⭐⭐（纯prompt优化，代码改动小）

---

### 🟢 壁垒等级 B — 体验差异化（易做见效快）

#### B1. 脚本生成后「一键跳转诊断」

**改动量：** 极小，Tab3结果区加一个按钮，点击后切到Tab8并自动填入脚本。

#### B2. 热搜条目「一键加入选题种子」

**改动量：** 小，热搜列表每项增加按钮，写入 `session_state["topic_seeds"]`。

#### B3. 选题结果「一键生成脚本」

**改动量：** 小，选题结果每项增加按钮，跳到Tab3并填入选题。

#### B4. 诊断结果导出为「优化报告」

**改动量：** 中，将诊断的六维评分+建议格式化为可下载的PDF/TXT。

#### B5. 标题优化结果「一键应用到脚本」

**改动量：** 小，选中标题后写入 `session_state["script_topic"]`，跳转Tab3。

## 五、总结

**当前工具的最大风险不是功能不够，而是功能之间没有形成闭环。**

用户可以在ChatGPT/Kimi里手动完成大部分单点功能。我们的壁垒来自于：

1. **闭环 > 单点** — 诊断→重写→再诊断 的迭代循环
2. **沉淀 > 一次性** — 公式库/创作历史越用越有价值
3. **联动 > 孤岛** — 一个工作台搞定所有事
4. **垂直 > 通用** — 平台算法级的专业适配

**建议立即启动 S1（诊断→一键重写）和 A1（跨Tab联动），这两项是投入产出比最高的技术壁垒。**
