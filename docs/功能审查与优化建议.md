# 功能审查与优化建议

> 对 AI短视频内容工作台 的全面代码审查

---

## 一、🐛 Bug（必须修复）

### Bug 1：占位符未替换
- **位置**：`app.py` 第112行、第209行
- **问题**：`your_wechat_id` 是占位符，用户会直接看到这个假文案
- **影响**：⭐⭐⭐ 影响专业度和转化率

### Bug 2：试用卡显示无意义的到期时间
- **位置**：`app.py` 第146-147行
- **问题**：试用卡(demo)的 `days=999`，激活后显示"到期时间：2028-12-27"，但试用卡是按次数过期的，这个日期完全没意义，容易让用户困惑
- **修复**：试用卡不显示到期时间

### Bug 3：云端每日限额形同虚设
- **位置**：`key_manager.py` 第156-165行
- **问题**：`usage_log`（每日用量）只存在本地 `keys.json` 和内存中。云端每次重启后 `usage_log` 重置为 `{}`，所以体验卡/月卡的"每天10次""每天200次"限制**在云端不生效**
- **影响**：⭐⭐ 月卡用户可能一天刷很多次（不过目前用户少，影响有限）
- **建议**：暂时可不改，后续如果用户量上来，在 Supabase 增加 daily_usage 字段

### Bug 4：未使用的 import
- **位置**：`key_manager.py` 第4行
- **问题**：`import hashlib` 从未使用
- **影响**：⭐ 不影响功能，但代码不够干净

## 二、⚠️ 体验问题（建议修复）

### UX 1：卡密过期后不能自动切换
- **现象**：试用卡用完3次后，侧边栏显示红色"试用次数已用完"，但输入框仍锁定，用户必须点"切换卡密"才能输入新卡
- **建议**：过期/失效时自动显示输入框 + 提示购买

### UX 2：生成失败不应扣次
- **位置**：`app.py` 第321、413、489行
- **问题**：`consume_usage` 在 `try` 块内，但位于 `results` 赋值之后。如果 AI 返回了结果但格式解析出错（下游报错），仍会扣次
- **建议**：将 `consume_usage` 移到确认结果成功显示之后

### UX 3：脚本生成按钮在输入框上方
- **位置**：`app.py` 第361-370行
- **问题**：用户先看到选题输入框，下面才是风格/平台/时长选项，再下面才有提交按钮。信息填写的视觉流向不够顺畅
- **建议**：按钮放到所有选项下面最后一行

### UX 4：热搜结果没有缓存提示
- **现象**：切到其他tab再切回来，热搜还在，但用户不知道这是之前的缓存
- **建议**：加个 `st.caption("上次抓取时间：10:30")`

## 三、🚀 优化建议（提升竞争力）

### 优化 1：加载状态体验
- **现状**：用 `st.spinner` 显示加载中，但没有进度反馈
- **建议**：可用 `st.progress` + 分步提示（"正在联想扩展…" → "正在AI打分…"）

### 优化 2：脚本导出功能
- **现状**：只有"一键复制"（st.code），没有下载
- **建议**：加 `st.download_button` 下载为 `.txt` 或 `.md` 文件

### 优化 3：分镜结果导出
- **建议**：分镜提示词一键导出为 Excel/CSV，方便发给剪辑师或导入工作流

### 优化 4：历史记录
- **现状**：刷新后所有生成结果丢失
- **建议**：用 `st.session_state` 保存最近5条记录，加个"历史记录"区域

### 优化 5：选题→脚本→分镜 一键流水线
- **现状**：需要在3个tab间手动切换和操作
- **建议**：增加"一键出片"模式，输入赛道后自动：选题→选最高分→生成脚本→生成分镜
